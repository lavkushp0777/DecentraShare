# DecentraShare - PHP Version

This is a PHP conversion of the original TypeScript/React DecentralShare application. The application provides decentralized file sharing using blockchain smart contracts and IPFS.

## Project Structure

```
DecentraShare/
├── public/                 # Web-accessible entry point
│   ├── index.php          # Main application controller
│   ├── .htaccess          # Apache routing configuration
│   ├── css/               # Stylesheets
│   │   └── style.css
│   └── js/                # Client-side JavaScript
│       └── app.js
├── src/                   # PHP source code
│   └── Utils/
│       ├── Web3.php       # Wallet/Web3 operations
│       ├── IPFS.php       # IPFS file handling
│       └── Contract.php   # Smart contract interactions
├── templates/             # PHP template files
│   ├── layout.php         # Main layout (App.tsx)
│   ├── upload.php         # File upload component
│   ├── my-files.php       # User files listing
│   └── shared-files.php   # Shared files listing
├── config/                # Configuration files
│   └── contracts.php      # Contract ABI and settings
├── composer.json          # PHP dependencies
├── .env.example           # Environment variables template
└── README.md              # This file
```

## Conversion Notes

### From TypeScript/React to PHP

1. **React Components → PHP Templates**
   - `src/App.tsx` → `templates/layout.php` + `public/index.php`
   - `src/components/FileUpload.tsx` → `templates/upload.php`
   - `src/components/FileViewer.tsx` → `templates/my-files.php` + `templates/shared-files.php`
   - `src/components/IPFSConfig.tsx` → Configuration in `public/js/app.js` and `.env`

2. **TypeScript Utilities → PHP Classes**
   - `src/utils/web3.ts` → `src/Utils/Web3.php`
   - `src/utils/ipfs.ts` → `src/Utils/IPFS.php`
   - `src/utils/contract.ts` → `src/Utils/Contract.php`

3. **Build Changes**
   - Vite build system removed (not needed for PHP)
   - Tailwind CSS loaded via CDN in templates
   - Frontend assets (CSS, JS) served from `public/` directory

4. **State Management**
   - React hooks → PHP Sessions (`$_SESSION`)
   - Component props → Templating variables
   - useEffect → Server-side processing + JavaScript event listeners

5. **MetaMask Integration**
   - Client-side signature verification via `personal_sign`
   - PHP session management for authenticated connections
   - Web3 provider detection in JavaScript

## Installation & Setup

### Prerequisites

- PHP 8.0+
- Composer
- Modern web server (Apache with mod_rewrite, Nginx, or built-in PHP server)
- MetaMask browser extension

### Installation Steps

1. **Install PHP Dependencies**
   ```bash
   composer install
   ```

2. **Setup Environment Variables**
   ```bash
   # Copy example to actual .env file
   cp .env.example .env
   
   # Edit .env and add your configuration
   ```

3. **Configure IPFS Provider**
   
   Choose one of the following IPFS providers and add credentials to `.env`:
   
   **Option A: Pinata (Recommended)**
   ```
   IPFS_PROVIDER=pinata
   PINATA_API_KEY=your_api_key
   PINATA_SECRET_KEY=your_secret_key
   ```
   
   **Option B: Web3.Storage**
   ```
   IPFS_PROVIDER=web3storage
   WEB3_STORAGE_TOKEN=your_token
   ```
   
   **Option C: Custom IPFS Node**
   ```
   IPFS_PROVIDER=custom
   IPFS_API_URL=http://localhost:5001
   IPFS_GATEWAY_URL=http://localhost:8080/ipfs/
   ```

4. **Configure Web3**
   ```
   CONTRACT_ADDRESS=0x70dD105c6D5F4be9aa803618abfCbBC5Fa1B1B82
   WEB3_RPC_URL=https://eth-mainnet.alchemyapi.io/v2/YOUR_KEY
   ```

5. **Start Development Server**
   
   Using PHP's built-in server:
   ```bash
   php -S localhost:8000 -t public
   ```
   
   Or with Apache:
   - Configure document root to point to `public/` directory
   - Ensure `mod_rewrite` is enabled

6. **Access Application**
   
   Open browser and navigate to:
   ```
   http://localhost:8000
   ```

## Architecture

### Frontend
- **HTML**: Generated by PHP templates
- **Styling**: Tailwind CSS (via CDN)
- **JavaScript**: Vanilla JS for wallet connection and UI interactions

### Backend
- **Language**: PHP 8.0+
- **Session Management**: PHP native sessions
- **HTTP Client**: Guzzle HTTP for API calls
- **Configuration**: Environment-based configuration

### Smart Contract Integration
- **Web3.js equivalent**: Web3 utility class for address validation
- **Contract Calls**: Direct RPC calls via custom implementation
- **Signature Verification**: Personal_sign verification in JavaScript

## Usage

### Connecting Wallet

1. Click "Connect Wallet" button
2. MetaMask will prompt for connection
3. Sign the verification message
4. Session is established on server
5. User can now upload and share files

### Uploading Files

1. Navigate to "Upload Files" tab
2. Drag and drop or click to select file (max 100MB)
3. Click "Upload to IPFS"
4. Progress bar shows upload status
5. File metadata is stored (blockchain submission would require signing)

### Managing Files

- **My Files**: View your uploaded files with options to share or delete
- **Shared Files**: View files shared with you by other users

## API Endpoints

All endpoints are accessible through `public/index.php`:

### POST /?action=verify_wallet_signature
Verify MetaMask signature and establish session
```json
{
  "address": "0x...",
  "signature": "0x...",
  "message": "..."
}
```

### POST /?action=upload
Upload file to IPFS (multipart form-data)
- Requires `wallet_address` in session
- File in `file` field

### POST /?action=disconnect
Disconnect wallet session

### POST /?action=toggle_dark_mode
Toggle dark mode preference

## Development

### Project Structure

```php
// Web3 utility - validate addresses, manage sessions
use DecentraShare\Utils\Web3;

// IPFS utility - handle file uploads to IPFS
use DecentraShare\Utils\IPFS;

// Contract utility - interact with smart contract
use DecentraShare\Utils\Contract;
```

### Adding New Features

1. Create utility class if needed in `src/Utils/`
2. Create template in `templates/`
3. Handle form submission in `public/index.php`
4. Add JavaScript event listener in `public/js/app.js`

### Error Handling

All exceptions are caught and returned as JSON responses:
```php
try {
    // Operation
} catch (Exception $e) {
    http_response_code(500);
    echo json_encode(['error' => $e->getMessage()]);
}
```

## Security Considerations

1. **Wallet Connection**: Uses MetaMask signature verification
2. **Session Management**: PHP sessions with `session_start()`
3. **File Validation**: Size limits and type checking
4. **Environment Variables**: Sensitive data in `.env` (not in repo)
5. **CORS**: Configure as needed for frontend requests

## Performance

- **Caching**: Implement Redis for session storage in production
- **File Compression**: Gzip enabled via `.htaccess`
- **Security Headers**: Added via `.htaccess`
- **Lazy Loading**: Files loaded on demand

## Limitations & Differences from TypeScript Version

1. **No Real-time Updates**: Web3 contract calls are mock implementations
2. **File Storage**: Uploaded files are not actually stored, only metadata
3. **Signature Verification**: Simplified client-side verification
4. **Browser Support**: Requires modern browser with MetaMask
5. **Direct Contract Calls**: Would need web3.php library for production
